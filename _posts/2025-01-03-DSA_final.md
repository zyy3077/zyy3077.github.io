## 第七章 图

### 1. 图的存储方法
- 相邻矩阵
- 邻接表
  - 顶点结点为(data | firstarc)，firstarc指向第一条以该顶点作为始点的边
  - 边结点为(adjvex | nextarc | info)，adjvex为这条边的终点，nextarc指向下一条以该顶点作为始点的边，info一般为边权值
  - 适用于稀疏图
  - 也有逆邻接表，即存储以该顶点为终点的链表
- 十字链表
  - 顶点结点为(data | firstoutarc | firstinarc)，后两者分别指向第一条以该顶点作为始点/终点的边
  - 边结点为(tailvex | tailnextarc | headvex | headnextarc | info)

### 2. 图的算法
- 周游算法
  - 深度优先
    - 一般采用递归实现，对\\(n\\)个顶点中的每个顶点至多调用一次DSF()；访问的时间代价为\\(O(n)\\)
    - 对于相邻矩阵，检查\\(n^2\\)个矩阵元素，时间代价为\\(O(n^2)\\)；总时间代价为\\(O(n^2 + n) = O(n^2)\\)
    - 对于邻接表，检查\\(e\\)条边，时间代价为\\(O(e)\\)；总时间代价为\\(O(e + n)\\)
  - 宽度优先
    - 一般采用优先队列实现
    - 时间复杂度与深度优先相同，只是顺序不同
- 拓扑排序
- 最短路径
  - Dijkstra算法
  - Floyd算法
- 最小生成树
  - Prim算法
  - Kruskal算法

## 第八章 内排序
### 1. 直接插入排序
- 主要思想：对每一个新加入的元素，在已排序序列中找到第一个不大于这个元素的元素，插入到这个位置之后（后面那些大于的，后移一个索引）
- 空间复杂度：只需要用一个临时变量存储当前插入的元素即可，\\(O(1)\\)
- 时间复杂度：对第\\(i,(i=0,1,...,n-1)\\)个插入的元素（也就是\\(a[i]\\)，因为\\(a[0]\\)不用插入）
  - 最差情况下（逆序）需要比较\\(i,(i=0,1,...,n-1)\\)次；数组内元素也要移动i次，加上当前元素保存在临时变量中的2次移动，移动次数共i+2次
    - 总比较次数\\(T = \sum_{i=1}^{n-1} i = \frac{n(n-1)}{2} = O(n^2)\\) 
    - 总移动次数\\(T = \sum_{i=1}^{n-1} (i+2) = \frac{(n+4)(n-1)}{2} = O(n^2)\\)
    - 总时间复杂度为\\(O(n^2)\\)
  - 最好情况下（正序），比较\\(1\\)次，移动\\(2\\)次
    - 总时间复杂度为\\(O(n)\\)
  - 平均情况下为\\(O(n^2)\\)
- 稳定性：因为比较的时候带等号，故保持原有顺序，**具有稳定性**

### 2. Shell排序
- 主要思想：先按较大间隔分为多个子序列，在子序列内部进行直接插入排序，逐渐减小间距直到1。这样能减小逆置对个数，从而减小时间复杂度
- 时间复杂度
  - 较好的增量序列可以使时间复杂度减小到\\(O(n^{3/2})\\)
  - 但达不到\\(O(n\log{n})\\)
- 稳定性：因为子序列中的移动跨越其他元素，故**不具有稳定性**

### 3. 直接选择排序
- 主要思想：在未排序的序列中找到最小的元素，与已排序序列之后的元素交换
- 时间复杂度：第\\(i,(i=0,1,...,n-1)\\)次选择，即选择最小元素与\\(a[i-1]\\)交换，从\\(a[i]~a[n-1]\\)共\\(n-i\\)次比较，\\(1\\)次交换
  - 总比较次数为\\(T = \sum_{i=1}^{n-1} (n-i) = \frac{n(n-1)}{2} = O(n^2)\\)
  - 总交换次数为\\(T = n-1 = O(n)\\)
  - 直接选择排序不依赖于原始数组的输入顺序，故最差/最好/平均时间复杂度均为\\(O(n^2)\\)
- 稳定性：由于交换时可能把原先的先后次序颠倒，故**不具有稳定性**

### 4. 堆排序
- 主要思想：利用堆来utilize上一次选择留下的大小信息，下面基于最大堆
  - 建立最大堆
  - 取出堆顶元素与数组末端元素交换，进行下滤操作，直到堆为空
- 时间复杂度
  - 建堆时间代价为\\(O(n)\\)
  - 下滤操作时间代价为\\(O(\log{n})\\)
  - 总时间代价为\\(O(n+n\log{n}) = O(n\log{n})\\)
  - 理论上最好/最差/平均时间复杂度均为\\(O(n\log{n})\\)
- 空间复杂度：一个临时变量，\\(O(1)\\)
- 稳定性：交换末端元素可能改变原有先后次序，故**不具有稳定性**
  
### 5. 冒泡排序
- 主要思想：不停地比较相邻元素，后者比前者小则向前冒泡，总能把最小的元素换到最前面；一次完整的冒泡中若没有交换，则提前终止
- 时间复杂度
  - 最差情况下（逆序），第\\(i,(i=0,1,...,n-1)\\)次冒泡（最后与\\(a[i-1]\\)比较）的比较次数为\\(n-i\\)次，交换次数为比较次数
    - 总交换次数/总比较次数为\\(T = \sum_{i=1}^{n-1} (n-i) = \frac{n(n-1)}{2} = O(n^2)\\)
  - 最好情况下（正序），一次完整冒泡后发现没有交换，提前终止
    - 总比较次数为\\(n-1=O(n)\\)，交换次数为\\(O(1)\\)
- 空间复杂度：使用了一个临时变量，\\(O(1)\\)
- 稳定性：由于只交换相邻元素，而且交换是不带等号的，故不会改变原有的次序，**具有稳定性**

### 6. 快速排序
- 主要思想
  - 选择一个轴值
    - 如何选择轴值？最差的情况是选择了最大或最小的元素，可以选取中间点
  - 将剩余序列分割成做子序列和右子序列，左子序列都不大于轴值，右子序列都大于轴值
    - 分割方法一：左右分别一个指针，左指针越过小于等于轴值的元素，右指针越过大于等于轴值的元素，分别停在两个元素上，交换；直到左右指针交叉
    - 改进的分割方法：从序列两端交替检查空闲位置
      - 分割前，先将轴值与最后一个元素交换，因为轴值保存在临时变量里，故最后一个位置是空闲位置，即右指针指向空闲位置
      - 左指针向右走，遇到元素值比右指针指向值大的，将左指针的元素值赋给右指针，此时左指针变为空闲位置
      - 右指针往左走，遇到元素值比左指针指向值小的，将右指针的元素值赋给左指针，此时右指针变为空闲位置
      - 重复直到左右指针交叉，最后停在的空闲位置放置轴值
  - 对左子序列和右子序列快排，直到剩0或1个元素
- 时间复杂度：由于比较次数比移动次数多得多，所以考虑比较次数，分割时的比较次数显然为\\(O(n)\\)
  - \\(T(n)=T(n-1)+c\cdot n,T(0)=T(1)=O(1)\\)
  - 最差情况下，分割的两部分分别长为\\(n-1,1\\)
    - \\(T(n) = T(1)+c\sum_{i=2}^{n}i = O(n^2)\\)
  - 最好情况下，两个子序列等长
    - \\(T(n) = 2T(n/2) + c\cdot n\\)
    - \\(T(n) = cn\log{n} + n = O(n\log{n})\\)
  - 平均情况下，总次数/总情况数，\\(T(n) = O(n\log{n})\\)
- 空间复杂度：递归算法，每次分割需要用到栈中的一个空间，最差为\\(O(n)\\) ，最好为\\(O(\log{n})\\)，平均为\\(O(\log{n})\\)
- 稳定性：因为分割存在的指针交换跨越多个元素，可能改变原有次序，故**不具有稳定性**

### 7. 归并排序
- 主要思想：将序列划分为两个子序列，分别递归进行归并排序，将这两个排好序的子序列归并为一个有序序列
  - 归并：每次比较子序列头，取较小的进入结果序列，其后的元素顶上来成为新的子序列头；直到其中一个子序列为空，另一个子序列的元素直接进入结果序列
- 时间复杂度
  - 划分子序列的时间\\(O(1)\\)
  - 子序列排序的时间\\(T(n/2)\\)
  - 归并时间\\(O(n)\\)
  - \\(T(n) = T(n/2) + c\cdot n \Rightarrow T(n) = O(n\log{n})\\)
- 空间复杂度：归并时需要一个临时数组来从排好序的子序列中取出头部，\\(O(n)\\)
- 稳定性：因为在归并时若元素值相同，按原有次序进入结果队列，故**具有稳定性**

### 8. 桶式排序
- 主要思想：当事先已经知道长为\\(n\\)的数组中元素值都在\\([0,m-1]\\)中时
  - count数组记录每个元素出现的次数
  - count[i] = count[i-1] + count[i]，记录不超过i的元素个数
  - 将TempArray[i]放到Array[count[TempArray[i] - 1]]，并将count[TempArray[i]]--
- 时间复杂度
  - 上述每一步为\\(O(n)\\)或\\(O(m)\\)
  - 总时间复杂度为\\(O(n+m)\\)
- 空间复杂度
  - TempArray为\\(O(n)\\)
  - count为\\(O(m)\\)
  - 总空间复杂度为\\(O(n+m)\\)
- 稳定性：从后向前放置元素，保持原有先后次序，故**具有稳定性**

### 9. 基数排序
- 主要思想：桶式排序的\\(m\\)较大时，分为多个部分进行比较
  - 高位优先
    - 先按十位分到10个桶，再将每个桶按个位分为10个小桶
    - 将100个桶里的元素按顺序收集起来
  - 低位优先
    - 先按个位分到10个桶，按顺序收集起来
    - 再按十位分到10个桶，按顺序收集起来
  - 这里的分配和收集指的是：
    - 按count分配桶
    - 按桶的顺序和桶内元素的顺序收集
- 时间复杂度
  - 顺序存储
    - \\(d\\)次调用桶式排序，时间代价为\\(O(d(n+r))\\)
  - 静态链：只修改指针，最后做一次索引排序\\(O(n)\\)
    - 分配和收集只需要求改记录的next指针
    - 处理一位时，分配时间\\(O(n)\\)，收集时间\\(O(n+r)\\)
    - 总时间代价\\(O(d(n+r)) = O(d\cdot n), d\geq \log_{r}{n} \Rightarrow T(n) = O(n\log{n})\\)
- 空间复杂度
  - 顺序存储
    - 每次分配和收集各移动\\(n\\)次
    - 每个队列也要按\\(n\\)分配空间
  - 静态链
    - \\(n\\)个待排记录和\\(r\\)个桶的头尾指针，\\(O(n+r)\\)
- 稳定性：类似桶式排序，**具有稳定性**
